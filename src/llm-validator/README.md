# llm-validator — 非決定的出力の階層化品質設計の題材(骨組み)

LLM など非決定的な生成器の出力を、**揺らぎを層で封じ込めて**信頼できる結果に変えるバリデータ。
出力が毎回変わるので、単一の期待値アサーションは効かない。品質を3層に分けて設計する。

## 3層の設計(なぜこの構造か)

| 層 | 役割 | 何をテストするか | reference |
|---|---|---|---|
| 入口層 | 入力検証を網羅 | 不正/境界入力を同値分割・境界値で弾く(決定的に検査できる) | blackbox-systematic |
| 中央層 | 性質 + 複数サンプリング合意 | 出力スキーマ強制、不変な性質(メタモルフィック)、N回サンプルの多数決合意 | ai-nondeterministic |
| 出口層 | 後段バリデーション | 判定を外部化(別ルール/別モデル)し、最終出力が制約を満たすことを保証 | ai-nondeterministic |

非決定要素は層の外へ追い出す: 入口と出口は決定的に検査でき、中央だけが揺らぐ。
中央の揺らぎは「単発の正解」でなく「性質」と「合意」で抑える。

## テストリスト(縦貫き完了)

- [x] 入口層: 入力スキーマ違反・境界(空、空白のみ、型違い、1000/1001字)を網羅的に拒否
- [x] 中央層(性質): 出力が常にスキーマ適合・不変条件(label∈許可集合、0≤agreement≤1)を満たす(`fc.property` で複数サンプル)
- [x] 中央層(合意): 同一入力を N 回サンプルし、多数決でラベル決定・合意率を算出。全無効なら throw
- [x] 出口層: 後段バリデータが閾値割れ・許可外ラベルを確実に捕捉(決定的アサーション)

実装: `src/llm-validator/validator.ts`(3層を `validateInput` / `classifyWithConsensus` / `assertValidLabel` で表現)。
テスト: `test/llm-validator.test.ts`(3層に1対1の `describe`)。`npm test` で全 green。
LLM 本体はフェイク(注入された純粋関数 `Classifier`)で差し替える(出力ベース検証、モック濫用しない。実 API は呼ばない)。
