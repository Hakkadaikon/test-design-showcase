# expr — 構造網羅とメタモルフィックの題材(骨組み)

四則演算 + 変数の式評価器(`1+2*x` を環境 `{x: 3}` で評価して `7`)。
分岐とループを持つ実装なので**ホワイトボックス網羅**が、出力の正解が言いにくい性質には**メタモルフィック**が効く。

## 振る舞いに、なぜこの手法を割り当てるか

| 振る舞い | 手法 | reference |
|---|---|---|
| パーサ/評価器の全分岐(演算子種別、括弧、優先順位)を通す | C0/C1、必要なら MC/DC | whitebox-coverage |
| 演算子の組合せ規則(加減乗除の優先・結合) | デシジョンテーブル | blackbox-systematic |
| `a+b == b+a`(加法可換)、`(a+b)+c == a+(b+c)`(結合) | メタモルフィックテスト | modern-generative |
| 未定義変数・ゼロ除算・空式 | 同値分割(無効クラス)+ 境界値 | blackbox-systematic |

## テストリスト骨組み

- [ ] C0/C1: 各演算子・括弧・優先順位の分岐を1つずつ通す `it.each`
- [ ] メタモルフィック: 可換・結合を `fc.property` で(値の絶対正解を用意せず関係で縛る)
- [ ] 異常系: 未定義変数 / ゼロ除算 / 構文エラーを throw で
- [ ] 境界: 深いネスト、長い式(ループ境界)

カバレッジ率は目的化しない。到達(C0/C1)は検証を保証しないので、強さは mutation(Stryker)で点検する想定。
実装(`expr.ts`)とテストはまだ骨組み。
